# Trending Paper Dashboard - 要件定義書

## 1. プロジェクト概要
PythonとStreamlitを使用し、Hugging FaceのTrending Papers（トレンド論文）情報を取得・表示するダッシュボードアプリケーションを開発する。研究者やAIエンジニアが最新のトレンドを効率的にキャッチアップできることを目的とする。

## 2. 機能要件

### 2.1 データ取得機能
- **データソース**: Hugging Face (Trending Papers)
- **取得内容**:
  - 論文タイトル
  - 要約（Abstract）または短い説明
  - 元記事（Hugging Faceの論文ページまたはArxiv）へのリンクURL
  - (オプション) 著者情報、公開日、サムネイル画像等があれば取得
  - **データ保存**: 取得したデータは日付ごとにローカル（JSON等）に保存し、再利用可能にする。

### 2.2 検索・フィルタリング機能
- **UIコンポーネント**: サイドバーに検索ボックスを配置する。
- **デフォルト動作**: 検索ボックスの初期値（デフォルト）は「Artificial Intelligence」とする。
- **検索対象**: 過去に取得・保存されたデータに対して検索を行う（オフラインでも閲覧可能にする）。
- **フィルタリング**:
  - 入力されたキーワードに基づいて、**保存済みのデータ**から論文をフィルタリングする。
  - **日付フィルタ**: 保存済みのデータから特定の日付（「今日」「過去の日付」）を選択して表示する。

### 2.3 表示機能
- **レイアウト**: カード型デザイン（Grid Layout等）を採用し、視認性を高める。
- **カード構成要素**:
  - **タイトル**: 論文のタイトルを目立たせて表示。
  - **要約**: 内容が十分に分かる程度の詳細な要約を表示する（横長レイアウトでも可）。
  - **リンクボタン**: 「元記事を読む」等のボタンを配置し、クリックで外部サイトへ遷移する。

## 3. 非機能要件
- **レスポンス**: データの取得・表示が許容範囲内の時間で行われること（API制限等の考慮）。
- **ユーザビリティ**: 直感的でシンプルなUI/UX。
- **保守性**: コードは適切にモジュール化し、可読性を保つ。

## 4. 技術スタック
- **言語**: Python 3.x
- **Webフレームワーク**: Streamlit
- **データ取得**:
  - `requests` + `BeautifulSoup` (スクレイピングの場合)
  - または `feedparser` (RSSがある場合)
  - または `huggingface_hub` API
  - ※実装時に最適な方法を選定
- **環境管理**: `venv` (標準ライブラリ)
- **パッケージ管理**: `pip` (requirements.txt)

## 5. 環境構築・納品物
- 仮想環境 (`venv`) の作成スクリプトまたは手順
- `requirements.txt` の提供
- ソースコード一式 (`app.py` 等)
