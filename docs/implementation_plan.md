# Trending Paper Dashboard 実装計画書

## ゴール
Streamlitを使用して、Hugging Faceのトレンド論文を表示するダッシュボードを構築します。データは日次でローカルにJSONとして保存し、検索は保存されたデータに対して行います。

## 変更案

### プロジェクトルート
#### [MODIFY] [app.py](file:///c:/Users/daich/machine_learning/12-trending_paper/app.py)
- **データ管理**:
  - `data/` ディレクトリを作成。
  - `fetch_and_save_daily_papers(date)`:
    - 指定された日付のJSONファイル (`data/YYYY-MM-DD.json`) が存在するか確認。
    - **存在しない場合**:
      - スクレイピング (`requests` + `BeautifulSoup`) で `https://huggingface.co/papers?date=YYYY-MM-DD` からデータを取得。
      - JSONファイルとして保存。
    - **存在する場合**:
      - JSONファイルを読み込む。
- **データ取得ロジック（スクレイピング）**:
  - `huggingface_hub` APIは検索に特化しているため、その日のトレンド一覧取得にはスクレイピングを使用する。
  - 取得項目: タイトル, リンク, 日付, サムネイルURL(任意)。
  - **個別要約取得**: 一覧に要約が含まれない場合、各論文ページにアクセスして要約を取得するか、または `huggingface_hub` APIでIDから詳細を取得する（今回は一覧取得を優先し、要約はAPI取得を補助的に使うハイブリッドも検討）。
  - **簡易実装**: まずは一覧ページから取れる情報で構成する。

### フォルダ構成
- `data/` : JSONファイル保存用

## 検証計画
1. **保存テスト**: アプリ起動後、特定の日付を選択し、`data/` にJSONが生成されるか確認。
2. **読み込みテスト**: アプリを再起動しても、保存済みのデータが表示されるか確認。
3. **検索テスト**: 保存されたデータに対してキーワード検索が機能するか確認。
